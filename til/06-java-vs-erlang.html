Erlang module vs Java Class

So, I am reading Joe Armstrong's thesis "Making reliable distributed systems in the presence of sofware errors" and in chapter 3, page 42, there is an erlang module introduction. The code is below, but I think we need more new programming languages, which let us express same idea in fewer lines of code. World need more programming language people.


An erlang module

    -module(math).
    -export([areas/1]).
    -import(lists, [map/2]).
    
    areas(L) ->
        lists:sum(
            map(
                fun(I) -> area(I) end,
            )).
    
    area({square, X}) ->
        X*X;
    area({rectangle,X,Y}) ->
        X*Y.

Wanna be Java program

    package org.j0.jnfeatures.records;

    import java.util.List;
    
    public class LikeErl {
        interface Shape { 
            double area();
        };
        
        record Square(double  side) implements Shape {
            public double  area() {
                return side * side;
            }
        }
        
        record Rectangle(double  length, double  width)   implements Shape{
            public double  area() {
                return length * width;
            }
        }
        
        public static double sumOfAreas(List<Shape> shapes) {
            if(shapes ==null || shapes.isEmpty()) {
                return 0.0;
            }
            return shapes.stream().mapToDouble(s -> getArea(s)).sum();
        }
        
        public static double getArea(Shape s) {
            return switch (s) {
                case Rectangle r -> r.area();
                case Square sqr  -> sqr.area();
                default          -> throw new IllegalArgumentException("Unrecognized shape");
            };
        }
        
        public static void main(String[] args) {
            Square s = new Square(2.5);
            Rectangle r = new Rectangle(2.3, 2.7);
            System.out.println(getArea(s));
            System.out.println(getArea(r));
            
            var totalAreas = sumOfAreas(List.of(s, r));
            System.out.println(totalAreas);
        }
    
    }
